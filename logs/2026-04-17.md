## Overnight Pretraining Runs (1M steps each)

Three experiments run sequentially to extend meta-pretraining to 1M steps and add a new Adam inner-loop variant. MAML and FOMAML used `meta_lr_inner=0.01` (up from `4e-5` in the 200k runs). The Adam-FOMAML used `meta_lr_inner=1e-4` to match the OGD Adam learning rate from streaming experiments. All three use the same LoRA-XS architecture (`dense_lora_xs`, rank=48, all 4 layers), full model trains together (backbone not frozen).

### New trainer: `adam_fomaml_tdmpc2`

Added `init_adam_fomaml_tdmpc2_trainer` to `max/trainers.py`. Identical to FOMAML except the inner adaptation step is a single Adam step from zero state rather than SGD. At t=1 bias-correction cancels, giving `r' = r - lr * grad / (|grad| + eps)` — approximately sign(grad). This matches the OGD Adam update used in streaming adaptation, so meta-pretraining and adaptation use the same optimizer geometry.

## Streaming Adaptation Experiments

Five experiments run sequentially (one at a time to avoid GPU contention) to compare online adaptation methods. All evaluate on HalfCheetah with `cheetah_mass_scale=0.1`, 3 seeds × 2 processes, 5 eval episodes, 10k steps, logged to `cheetah_meta_1m_streaming`.

All use `dense_lora_xs` dynamics (rank=48, 4 layers, frozen backbone, frozen subspace). Only the R (coefficient) matrices adapt at test time. Planner: MPPI with `num_iterations=6, batch_size=512, horizon=3` for all experiments.

### Experiments

**stream_ogd_loraxs_base** — baseline: does LoRA-XS adaptation help even without meta-pretraining?
- Base model: vanilla TDMPC2 (`cheetah_tdmpc2/20260415_231700/step_1000000.pkl`)
- Trainer: `ogd` (Adam), `lr=1e-4`, `horizon=1`, `temporal_decay=0.5`

**stream_gd_loraxs_maml** — MAML inner-loop SGD adaptation
- Base model: MAML (`cheetah_maml/20260416_224807/step_1000000.pkl`)
- Trainer: `gd_lora_xs`, `meta_lr_inner=0.01`

**stream_gd_loraxs_fomaml** — FOMAML inner-loop SGD adaptation
- Base model: FOMAML (`cheetah_fomaml/20260417_022600/step_1000000.pkl`)
- Trainer: `gd_lora_xs`, `meta_lr_inner=0.01`

**stream_ogd_loraxs_fomaml_adam** — Adam-FOMAML with OGD Adam adaptation
- Base model: Adam-FOMAML (`cheetah_fomaml_adam/20260417_055420/step_1000000.pkl`)
- Trainer: `ogd`, `lr=1e-4`

**stream_ekf_loraxs** — EKF adaptation on vanilla TDMPC2 base (sanity check)
- Base model: vanilla TDMPC2 (`cheetah_tdmpc2/20260415_231700/step_1000000.pkl`)

## EKF Planner Investigation

Forensic investigation showed the original EKF-LoRA-XS success (614 ± 49, April 13–14) used a **weak planner** (`num_iterations=1, batch_size=64`), while the April 17 runs above used the strong planner (`num_iterations=6, batch_size=512`). Code and math are identical between f3c5e8f and current. The strong planner masks model-quality differences (robust to wrong dynamics via brute-force sampling) and amplifies multi-step rollout errors from imperfect R adaptation — causing some seeds to degrade.

### Test 1: cheetah_tdmpc2 model + weak planner (1×64)

Config: `stream_ekf_loraxs.json`, pretrained `cheetah_tdmpc2/20260415_231700/step_1000000.pkl`, planner `num_iterations=1, batch_size=64`. Wandb: `cheetah_meta_1m_streaming`, runs `cheetah_tdmpc2_p{1,2}_{1,2,3}`.

| Seed | eval/episode_reward |
|---|---|
| p1_1 | 646.8 |
| p1_2 | 645.5 |
| p1_3 | 638.9 |
| p2_1 | 758.6 |
| p2_2 | 623.8 |
| p2_3 | 721.2 |
| **Mean ± std** | **672 ± 50** |

**Result: hypothesis confirmed.** Weak planner recovers strong adaptation performance (matches/exceeds prior 614 ± 49). The planner change, not any code or model difference, was the cause of the April 17 regression.

### Test 2: less_features model + weak planner (1×64)

Config: `stream_ekf_loraxs.json`, pretrained `cheetah/20260323_less_features/final.pkl`, planner `num_iterations=1, batch_size=64`. Wandb: `cheetah_meta_1m_streaming`.

| Seed | eval/episode_reward |
|---|---|
| p1_1 | 571.3 |
| p1_2 | 604.0 |
| p1_3 | 661.9 |
| p2_1 | 592.7 |
| p2_2 | 536.9 |
| p2_3 | 599.1 |
| **Mean ± std** | **594 ± 38** |

**Result: matches original (614 ± 49).** The pretrained model makes no meaningful difference — both models recover strong adaptation with the weak planner.

### Test 3: less_features model + strong planner (6×512) — done

Config: same but planner `num_iterations=6, batch_size=512`. Goal: confirm strong planner degrades EKF adaptation even with the original model.

| Seed | eval/episode_reward |
|---|---|
| p1_1 | 481.0 |
| p1_2 | 552.8 |
| p1_3 | 495.1 |
| p2_1 | 386.9 |
| p2_2 | 661.0 |
| p2_3 | 527.8 |
| **Mean ± std** | **517 ± 83** |

**Result: partial confirmation.** Strong planner reduces performance vs. weak planner (517 ± 83 vs. 594 ± 38), and all 6 seeds still show improvement from initial reward. Notably, less_features + strong planner (517) outperforms cheetah_tdmpc2 + strong planner (the original degrading runs at ~270). This suggests the pretrained model matters when using the strong planner — less_features R-matrices may be better initialized for EKF adaptation.

## Extreme Mass Shift Experiments (mass_scale=0.01)

Working hypothesis: `cheetah_tdmpc2` starts too close to the ceiling at `mass_scale=0.1` — EKF has little residual signal to work with and its R updates degrade multi-step rollouts under the strong planner. `less_features` starts lower so EKF has more room. If true, forcing `cheetah_tdmpc2` into an even harder regime (`mass_scale=0.01`, 100× lighter than training) should close the gap between models and let EKF adapt successfully for both.

Both experiments: strong planner (6×512), EKF trainer (`lr=10, init_cov_scale=0.01`), 10k steps, 3 seeds × 2 processes, logged to `cheetah_meta_1m_streaming`.
