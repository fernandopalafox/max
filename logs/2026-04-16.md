## Bug Fixes in LoRA-XS dynamics

Three bugs fixed in `max/dynamics.py`:

1. **Backbone not loaded from LoRA-XS checkpoints**: the loading code only handled the dense Flax `"params"` key, not the `"backbone"` key from LoRA-XS checkpoints. Backbone was silently randomly re-initialized on every load.
2. **R not loaded from checkpoint**: pretrained R matrices were discarded and re-initialized near zero, discarding the trained adapter.
3. **SVD Σ scaling missing**: SVD init used only the orthonormal U vectors for P, dropping the singular values. The original LoRA-XS parametrization was `W + U @ diag(Σ) @ R @ V^T`, so R updates are scaled by singular value magnitude. Fixed by absorbing Σ into P: `P = U @ diag(Σ)`.

All streaming runs: mass_scale=0.1, 10k steps, latest sampler (batch_size=1), encoder/critic/policy/reward frozen, only dynamics R adapts.

### Key findings

- All GD and OGD runs with the MAML/FOMAML/nometa 200k checkpoints showed no meaningful adaptation improvement. Root cause: backbone loading bug (fix 1 above) meant backbone was randomly initialized for all those runs — results are invalid.
- OGD on `less_features` dense model with fresh LoRA-XS (no Σ) showed some adaptation but was inconsistent.
- OGD on `less_features` dense model with fresh LoRA-XS (with Σ fix) is the current clean baseline being validated (`ogd-less-features-svd` runs).
- LR sweep (1e-5 to 1.0, plain GD, 3 seeds each) showed no improvement — likely because backbone loading was broken for all those runs.
